# pcap2llm v0.1.0 Release Notes

Release date: 2026-06-22

`pcap2llm` 0.1.0 is the first release line for the local CLI and Web GUI
workflow. It is aimed at focused telecom PCAP troubleshooting, deterministic
artifact generation, privacy-controlled sharing, and external LLM handoff.

## Highlights

- CLI workflow for `inspect`, `discover`, `recommend-profiles`, `analyze`,
  `visualize`, and LLM-oriented handoff commands.
- Built-in profile families for LTE/EPC, 5G core, VoLTE/VoNR, legacy 2G/3G,
  transport, DNS, and core name resolution.
- Privacy profiles for local work, internal sharing, production-safe masking,
  and external LLM handoff.
- Stable `detail.json` and `summary.json` artifact contracts with schema docs.
- Optional local Web GUI for uploads, discovery, analysis reruns, artifact
  review, profile management, dashboard views, and flow previews.
- Flow model and SVG rendering for signaling-sequence orientation.
- Deterministic network-element detection with exact, subnet, hostname, and
  protocol/port evidence.

## Hardening Since The April Baseline

- Same-origin POST checks protect the local Web GUI against cross-origin form
  posts.
- Binding outside localhost requires `PCAP2LLM_WEB_ALLOW_REMOTE=1`.
- Capture uploads and helper-file uploads have separate size limits.
- Analysis limits reject invalid negative values and too-small flow widths.
- Sensitive sidecars such as `pseudonym_mapping.json` and `vault.json` require
  explicit confirmation for direct download.
- Capture hashing now streams data instead of reading complete files into
  memory.

## Known Limits

- The Web GUI is local-first. It does not provide authentication,
  authorization, or rate limiting.
- Remote deployment is unsupported without additional hardening.
- CI covers one Ubuntu-packaged TShark line, not a full Wireshark/TShark
  compatibility matrix.
- Large captures still need focused display filters. Pass 1 scans the full
  filtered capture even when `detail.json` output is bounded.
- PyPI publishing is not part of this release.
